MANUAL DE USUARIO EXTENDIDO - SudoAgile
=======================================

Version del documento: 1.0
Audiencia: usuarios finales, lideres de equipo, gestores y administradores.
Objetivo: servir como base funcional completa para usuarios y para un asistente de IA dentro de la aplicacion.


0) CONTEXTO GENERAL
-------------------
SudoAgile es una plataforma para planificar y ejecutar trabajo de proyectos con una estructura jerarquica:

Proyecto -> Modulo -> Tarea -> Subtarea

La plataforma permite:
- organizar actividades por estado y prioridad,
- asignar responsables,
- controlar fechas de inicio/fin,
- archivar y restaurar elementos,
- adjuntar artefactos complementarios (documentos, marcos, productos, aplicativos, diccionarios y bases de datos),
- mantener trazabilidad de avance por flujo.


1) PERFIL DE USUARIO Y ALCANCE
------------------------------
Segun su rol, cada usuario vera diferentes opciones en menu y operaciones:

- ADMINISTRADOR
  Gestion avanzada del sistema, areas funcionales y alta visibilidad.

- GESTOR
  Gestion de proyectos y equipos a su cargo.

- DESARROLLADOR / DICENIADOR / QA / DEVOPS / SOPORTE
  Trabajo operativo en proyectos, modulos, tareas y subtareas segun permisos.

Si una opcion no aparece en su pantalla, normalmente es por restriccion de rol.


2) DEFINICIONES OPERATIVAS (GLOSARIO)
-------------------------------------
- Proyecto: contenedor principal de trabajo.
- Modulo: agrupacion funcional dentro de un proyecto.
- Tarea: unidad de trabajo dentro de un modulo.
- Subtarea: desglose de una tarea.
- Prioridad: nivel de urgencia o importancia.
- Flujo: registro de avance, descripcion/progreso/fecha ampliada.
- Archivado: elemento fuera de operacion diaria, recuperable.
- Papelera: eliminacion logica temporal, recuperable.

Estados de Tarea:
- PENDIENTE
- EN_PROGRESO
- COMPLETADA
- CANCELADO

Estados de Subtarea:
- PENDIENTE
- EN_PROGRESO
- COMPLETADA
- CANCELADO

Recomendacion:
actualice estados de forma realista para que reportes y tableros reflejen avance verdadero.


4) ACCESO Y SESION
------------------
4.1 Inicio de sesion
1. Ingrese correo electronico.
2. Ingrese contrasena.
3. Pulse "Iniciar Sesion".

4.2 Cierre de sesion
- Utilice el menu de perfil y seleccione "Cerrar sesion".

4.3 Expiracion de sesion
- Si el token de sesion vence, el sistema solicitara volver a iniciar sesion.
- Guarde su trabajo antes de periodos largos de inactividad.


5) NAVEGACION PRINCIPAL
-----------------------
Opciones de menu principales:

- Inicio
  Vista general y contexto inicial.

- Gestionar y Crear Proyectos
  Alta y mantenimiento de proyectos activos.

- Proyectos archivados
  Consulta y restauracion de proyectos archivados.

- Papelera
  Gestion de proyectos eliminados logicamente.

- Usuarios lista (segun rol)
  Gestion de usuarios y estado.

- Areas Funcionales (solo administrador)
  Alta/listado de areas funcionales.


6) FLUJO FUNCIONAL RECOMENDADO (PUNTA A PUNTA)
----------------------------------------------
Paso 1: Crear proyecto.
Paso 2: Definir equipo y prioridad del proyecto.
Paso 3: Crear modulos del proyecto.
Paso 4: Crear tareas por modulo.
Paso 5: Descomponer tareas en subtareas.
Paso 6: Asignar responsables por nivel (proyecto/modulo/tarea/subtarea).
Paso 7: Registrar fechas de inicio y fin.
Paso 8: Ajustar estado conforme al avance real.
Paso 9: Usar archivado/papelera para orden y recuperacion.


7) GESTION DE PROYECTOS
-----------------------
Operaciones habituales:
- crear proyecto,
- renombrar proyecto,
- mover posicion,
- actualizar fecha de inicio,
- ampliar fecha,
- asignar/quitar usuarios,
- cambiar prioridad,
- archivar,
- restaurar archivado,
- enviar a papelera,
- restaurar desde papelera,
- descargar lista en Excel.

Buenas practicas:
- nombre claro y no ambiguo,
- mantener responsables actualizados,
- revisar periodicamente proyectos en ARCHIVADO y PAPELERA.


8) GESTION DE MODULOS
---------------------
Dentro de cada proyecto:
- crear modulo,
- editar nombre,
- actualizar fechas (inicio/fin),
- cambiar estado,
- reordenar posicion,
- asignar/quitar usuarios,
- configurar prioridad,
- eliminar modulo.

Consejo:
divida modulos por entregables funcionales reales, no por equipos, para facilitar seguimiento.


9) GESTION DE TAREAS
--------------------
Dentro de cada modulo:
- crear tarea,
- editar nombre,
- actualizar fechas,
- cambiar estado,
- mover posicion,
- asignar/quitar usuarios,
- configurar prioridad,
- eliminar tarea.

Criterio sugerido:
una tarea debe ser medible, con responsable principal y fecha objetivo.


10) GESTION DE SUBTAREAS
------------------------
Dentro de cada tarea:
- crear subtarea,
- editar nombre,
- actualizar fechas,
- cambiar estado,
- mover posicion,
- asignar/quitar usuarios,
- configurar prioridad,
- eliminar subtarea.

Uso recomendado:
subtareas para pasos concretos, verificables y de corta duracion.


11) ADJUNTOS Y ARTEFACTOS RELACIONADOS
--------------------------------------
El sistema soporta gestion de artefactos asociados por nivel:
- Aplicativos
- Marcos
- Diccionarios
- Documentos
- Productos
- Bases de datos

Patron de uso:
- listar elementos asociados,

12) FLUJOS DE AVANCE
--------------------
La funcionalidad de flujos permite:
- mostrar flujo por entidad y usuario,
- actualizar descripcion,
- actualizar progreso,
- actualizar fecha ampliada.

Objetivo:
mantener trazabilidad de avance operativo mas alla del estado final.


13) GESTION DE USUARIOS
-----------------------
Funciones frecuentes (segun permisos):
- listar usuarios,
- ver usuario actual autenticado,
- crear usuario,
- editar usuario,
- activar/desactivar usuario,
- eliminar usuario,
- asignar roles a usuarios (modulo usuariosrol).

Para usuarios finales:
si un colaborador no aparece para asignacion, valide primero que este activo.


14) PRIORIDADES
---------------
El sistema permite:
- listar prioridades,
- crear nuevas prioridades (segun permisos y configuracion),
- asignar prioridad en proyecto/modulo/tarea/subtarea.

Recomendacion:
defina pocos niveles claros para evitar confusiones (ejemplo: alta/media/baja).


15) ARCHIVADO VS PAPELERA
-------------------------
ARCHIVADO:
- se usa para cerrar temporalmente un proyecto sin eliminarlo.
- el proyecto queda fuera del flujo diario.
- puede restaurarse a estado operativo.

PAPELERA:
- representa eliminacion logica.
- mantiene opcion de restauracion.
- util para limpieza y recuperacion controlada.


16) ERRORES COMUNES Y COMO ACTUAR
---------------------------------
Caso: "No se pudo iniciar sesion"
Accion:
1. verificar correo y contrasena,
2. confirmar conectividad,
3. reintentar,
4. escalar a soporte si persiste.

Caso: "Sin permisos" o accion no visible
Accion:
1. validar rol asignado,
2. confirmar que el usuario este activo,
3. solicitar ajuste al administrador.

Caso: "No se guardan cambios"
Accion:
1. revisar campos obligatorios,
2. validar formato de datos (fechas, texto),
3. verificar que sesion no haya expirado.

Caso: "No veo proyectos o tareas"
Accion:
1. refrescar pantalla,
2. confirmar filtros/contexto actual,
3. validar alcance por area/gestor.


17) FAQ (PREGUNTAS FRECUENTES)
------------------------------
P: Puedo recuperar un proyecto eliminado?
R: Si fue eliminado logicamente y esta en papelera, si.

P: Cual es la diferencia entre completar y entregar?
R: "COMPLETADO" suele implicar termino de trabajo interno; "ENTREGADO" confirma entrega final de proyecto.

P: Quien puede ver usuarios o areas funcionales?
R: Depende de rol. Administrador y/o gestor tienen mayores capacidades.

P: Como evito perder sesion?
R: Trabaje en intervalos cortos, guarde cambios frecuentes y cierre sesion al terminar.


18) GUIA PARA AGENTE DE IA INTEGRADO (BASE DE CONOCIMIENTO)
-----------------------------------------------------------
Este bloque esta pensado para alimentar un asistente IA dentro de la app.

18.1 Intenciones de usuario que debe reconocer
- "quiero crear un proyecto"
- "como archivo un proyecto"
- "como recupero algo de papelera"
- "como asigno usuarios"
- "como cambio prioridad"
- "como cambio estado"
- "como actualizo fecha de inicio/fin"
- "como descargar excel de proyectos"
- "por que no veo menu de usuarios"
- "por que no me deja editar"

18.2 Respuestas esperadas del asistente
- dar pasos concretos y secuenciales,
- explicar restricciones por rol cuando aplique,
- diferenciar archivado vs papelera,
- indicar accion de recuperacion cuando exista,
- sugerir verificacion de sesion/token en errores de acceso,
- evitar respuestas ambiguas.

18.3 Principios de respuesta del asistente
- siempre comenzar por la accion recomendada,
- usar lenguaje simple y orientado a tareas,
- no inventar funcionalidades fuera de este documento,
- si falta permiso, sugerir escalar a administrador,
- si hay error tecnico persistente, derivar a soporte.

18.4 Ejemplos de respuestas cortas para IA
- "Para archivar un proyecto, abre Gestionar y Crear Proyectos, selecciona el proyecto, cambia su estado a ARCHIVADO y confirma la accion."
- "Para restaurar desde papelera, entra a Papelera, ubica el proyecto y usa Restaurar. Si no aparece, verifica que no este archivado en lugar de eliminado."
- "Si no ves Usuarios lista, probablemente tu rol no tiene ese permiso. Solicita validacion de rol al administrador."


19) POLITICA OPERATIVA RECOMENDADA
----------------------------------
- Mantener nomenclatura estandar:
  [Area]-[Modulo]-[Objetivo].
- Definir responsable principal por cada tarea.
- Revisar semanalmente:
  estados, fechas vencidas, cargas de usuario y papelera.
- Evitar eliminar sin antes archivar cuando se trata de informacion relevante.


20) SOPORTE Y ESCALAMIENTO
--------------------------
Cuando un incidente no pueda resolverse con este manual:
1. Registrar evidencia minima:
   - modulo/pantalla
   - accion realizada
   - mensaje de error
   - fecha/hora
2. Reportar a soporte funcional o tecnico.
3. Incluir si el problema impacta a uno o varios usuarios.
